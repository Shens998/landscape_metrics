# 分块后端边界条件验证强化设计

> 状态：用户已确认
>
> 日期：2026-07-16

## 目标

将超大分类 GeoTIFF 的精确、内存受控分块计算路径，在目前的跨分块连接测试之外，进一步验证 NoData、显式背景、非方形像元、随机分类格局与临时文件清理。验收标准是分块与内存路径产生完全相同的 patch、class 和 landscape 表；临时目录行为不污染调用者目录。

## 范围与不变量

- 不增加指标、公开 API、CLI、并发、自动块大小或新的输入格式。
- 不改变 21 个冻结 v0.1 指标、稳定列名、4/8 斑块连通规则、共享边规则、NoData 排除规则或背景类别规则。
- 仅在新测试显示真实差异时修改生产代码；修复限于造成差异的最小范围。
- 继续使用单进程、磁盘映射标签、磁盘并查集、窗口 halo 和 SQLite 按块聚合；不得通过完整数组读取规避分块计算。

## 验证层次

### 规则矩阵

构造同时含显式背景 `0`、NoData `-1` 和多个有效类别的 GeoTIFF，使用不同于默认值的像元宽高（20 m × 30 m）。在 connectivity 4/8 和块大小 `(1, 2)`、`(2, 1)`、`(2, 2)` 下，将三个公开结果表与内存路径逐列、逐行严格比较。输入 GeoTIFF 的 NoData 元数据明确为 `-1`。

### 随机性质测试

用固定 Hypothesis 策略生成很小的整数分类栅格，类别集合包含 `-1`、`0`、`1`、`2`。随机选择合法块大小与 4/8 连通规则。每个样本写成临时 GeoTIFF，比较内存与分块的 patch/class/landscape 表；测试次数保持有限，以避免将面向大数据的实现测试变为耗时基准。

### 临时目录生命周期

在成功计算后，调用者提供的临时目录只保留原有输入文件，不保留 `landscape-metrics-*` 私有目录。对异常路径，已有临时空间预检测试继续保证在工作目录创建前失败；本轮补充计算期间异常的清理验证仅在能够通过公开 API 稳定触发异常时加入。

## 验收

1. 规则矩阵的 6 组配置及随机样本均与内存路径严格一致。
2. 分块结果保留 `execution_path="chunked"` 与请求的块大小元数据。
3. 成功路径不在调用者临时目录留下私有工作目录。
4. 所有测试、Ruff 与 mypy 通过；文档进度记录反映新增覆盖范围。
