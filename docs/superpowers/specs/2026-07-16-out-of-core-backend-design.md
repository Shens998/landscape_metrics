# Open Landscape Metrics｜外存分块后端设计修订

> 状态：用户已确认，待书面审阅。
>
> 日期：2026-07-16
>
> 适用范围：v0.1 的超大单波段 LULC GeoTIFF 精确计算路径。

## 1. 修订原因

内存路径的 `Topology` 对象保存完整分类数组、有效掩码和标签数组，适用于中小型栅格。将分块路径也实现为同一对象，会在大 GeoTIFF 上重新占用全图内存，违背“有限单机内存受控处理超大文件”的目标。

因此，分块路径不返回 `Topology`。它直接产生斑块、类别和景观三级的表格结果与完整运行元数据。

## 2. 后端边界与 API

当 `Landscape.from_geotiff(..., tile_shape=(rows, cols), tempdir=...)` 接收到 `tile_shape` 时，实例使用 `ChunkedBackend`。不提供 `tile_shape` 时，维持现有内存后端。

两个后端暴露相同的三个公开方法：

- `.patch_metrics() -> MetricResult`
- `.class_metrics() -> MetricResult`
- `.metrics() -> MetricResult`

二者的指标值、列顺序、排序规则、NoData 规则、边界规则和连通规则必须严格一致。唯一预期差异是元数据中的 `execution_path`（`memory` 或 `chunked`）及块大小、临时目录版本和临时磁盘预算。

## 3. 流式精确算法

### 阶段 A：局部标签

1. 逐个 Rasterio window 读取分类值；任何时刻只持有当前块及其必要边界条带。
2. 块内按类别运行与内存路径相同的 4/8 邻域标签算法。
3. 给局部标签加全局递增偏移，写入磁盘映射的 `label.memmap`；不将完整标签驻留 RAM。
4. 记录每个暂定标签的类别值和像元数到 SQLite 临时数据库，使用事务批量写入。

### 阶段 B：跨块连通合并

1. 只读取相邻块的公共边界标签与类别条带，不扫描所有块内部。
2. 4 邻域仅合并跨公共边直接相接、类别相同的暂定标签。
3. 8 邻域额外比较公共边界的上下或左右偏移一格的像元，以及四块交汇处的对角连接。
4. 并查集父指针以磁盘映射数组存储；完成后将根标签写入 `root_labels.memmap`。

### 阶段 C：流式聚合

1. 再次逐窗口读取根标签与原始分类值；为当前窗口读取一像元 halo，以便在不保留全图的前提下检查四个侧邻。
2. 每个有效像元对四个侧邻计算周长：景观外框（若启用）和不同的有效类别均计入；NoData 邻边既不形成边也不计入面积或邻接。相同类别的共享边只在右/下方向计数一次，用于同类邻接统计。
3. 每个窗口以 NumPy 按根标签批量聚合像元数、面积、周长、同类共享边数、坐标一阶矩和二阶矩。
4. 批量 upsert 到 SQLite 斑块表；不逐像元执行 SQLite 写入。
5. 以 `sum_x`、`sum_y`、`sum_x2`、`sum_y2` 计算回转半径，不保留每个斑块的像元坐标。
6. 从 SQLite 读取有序斑块摘要，生成与内存后端列定义一致的 patch 表；由该表和类别级中间量生成 class、landscape 表。

## 4. 性能与资源约束

- 默认单进程、确定性执行；v0.1 不并发写 SQLite 或共享并查集。
- 通过窗口向量化、边界专用扫描、批量 SQLite 事务和顺序 memmap 访问降低 CPU 与 I/O 放大。
- 不调用 `np.unique`、`ndimage.label` 或布尔掩码于全图；它们只在单个块内运行。
- 临时磁盘预检使用保守预算：标签和根标签各 8 bytes/像元，父指针最多 8 bytes/暂定标签，SQLite 与工作空间预留至少 24 bytes/像元；合计按 **48 bytes/像元** 估算。空间不足时抛出 `TemporaryStorageError`，不开始部分计算。
- 块大小由用户明确设置；元数据记录 `tile_shape`。自动块大小不进入 v0.1，以免不同环境产生不可预期的内存峰值。
- 临时目录必须位于本地可写磁盘，计算成功或失败后均由 `TemporaryDirectory` 清理。

## 5. 正确性与性能验收

1. 对 4 与 8 邻域、至少三种块大小，分块与内存的 patch/class/landscape DataFrame 必须完全相同。
2. 覆盖跨水平边界、垂直边界、四块交汇对角连通、NoData 边界、显式背景类别和非方形像元。
3. 临时空间不足在任何文件创建前失败，并给出预算、可用空间和建议目录。
4. 基准测试记录大合成栅格下的块大小、总耗时、峰值 RSS、临时磁盘峰值和结果行数。
5. 文档明确分块路径是精确算法，不是近似或抽样；若任一指标不能满足严格一致性，不得对该指标启用分块模式。

## 6. 明确不做的事项

- v0.1 不加入 Dask、分布式调度、GPU、移动窗口、矢量输入或自动块大小。
- v0.1 不支持需要全局距离变换或图论求解的指标。
- v0.1 不为速度牺牲内存上界、确定性或与内存路径的一致性。
